# Week 9: The Application Layer

## Overview

Week 9 focused on web application security and the vulnerabilities most commonly exploited in modern attacks. Through hands-on labs, I explored how weaknesses in web applications and APIs can lead to unauthorized access, data exposure, and account compromise. The week emphasized identifying, testing, and documenting common web vulnerabilities while understanding the defensive controls required to mitigate them.

## Skills Developed

* Web application reconnaissance
* SQL Injection (SQLi) testing
* Cross-Site Scripting (XSS) analysis
* Cross-Site Request Forgery (CSRF) concepts
* API security assessment
* Burp Suite traffic interception
* HTTP request and response analysis
* Authorization testing
* Broken Object Level Authorization (BOLA/IDOR) assessment
* Web vulnerability reporting
* Security testing methodology
* Risk assessment and remediation planning

## Labs Completed

### Day 1: The Data Exfiltration

Conducted SQL injection testing against vulnerable web applications to identify authentication weaknesses, enumerate database structures, and assess the impact of insecure input handling.

**Artifact:** `sqli_report.txt`

### Day 2: The Poisoned Browser

Performed web application testing focused on Cross-Site Scripting (XSS) and Cross-Site Request Forgery (CSRF) vulnerabilities. Evaluated how improper input validation and session management weaknesses can affect application security.

**Artifact:** `xss_payloads.txt`

### Day 3: The Invisible Logic

Used Burp Suite to intercept and analyze API traffic, evaluate authorization controls, and assess application logic vulnerabilities. Tested API endpoints for improper access controls and documented findings.

**Artifact:** `api_audit.log`

### Day 4: Operation Omni-Portal

Completed a comprehensive web application security assessment by evaluating a simulated enterprise application for multiple vulnerability classes, documenting findings, and producing a professional assessment report.

**Artifact:** `OmniPortal_Assessment.md`

## Key Takeaways

This week strengthened my understanding of application-layer security and the risks associated with insecure coding practices, weak authorization controls, and improper input validation. Through web application and API security testing, I gained practical experience identifying common vulnerabilities, analyzing business logic flaws, and documenting security findings in a structured and professional manner.
