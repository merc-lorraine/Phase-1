4. REMEDIATION
* What is the primary defense against this attack?
Answer: [The primary defense against SQL injection is the use of parameterized queries or prepared statements which treats all user input strictly as non-executable data. Prepared statements use parameterized placeholders to "lock" the query’s intent before the data is ever processed. Parameterized queries ensure that even if an attacker enters malicious SQL commands, or SQL injection, the database engine will interpret it as literal text rather than instructions.]
