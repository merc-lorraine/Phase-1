1. REFLECTED XSS
* Payload Used: [Standard HTML: <h1>Hacked</h1>; JavaScript Injection: <script>alert('XSS')</script>]
* Concept: Why does this attack disappear when you refresh the page?
Answer: The server does not save the search query to the database. It is simply echoes what is sent. When the page is refreshed, the browser sends a new request to the server. Unless a new request also contains the payload in the URL or data, the server sends back a clean version of the page.

* Malicious URL Crafted: [<img src=http://192.168.56.103:8081/transfer?to=Attacker&amount=5000>]
* Concept: What is the primary defense against CSRF?
Answer: [The primary defense against CSRF is Anti-CSRF Token, or Synchronizer Token. It is a unique, cryptographically strong, and unpredictable random string generated by server-side application to prevent CSRF.
